## Source
- Issuer: U.S. Department of Labor, Employee Benefits Security Administration (EBSA)
- Public-use research files: https://www.dol.gov/agencies/ebsa/about-ebsa/our-activities/public-disclosure/foia/form-5500-datasets
- Direct ZIP base: `https://www.askebsa.dol.gov/FOIA%20Files/{year}/Latest/F_{NAME}_{year}_Latest.zip`
  where NAME тИИ {`5500`, `SCH_A`, `SCH_C`, `SCH_D`, `SCH_G`, `SCH_H`, `SCH_I`, `SCH_R`, `SCH_MB`, `SCH_SB`}
- No authentication needed. (Note: hit `www.askebsa.dol.gov` directly тАФ the apex `askebsa.dol.gov` 301-redirects with malformed Location headers that hang `urllib`.)

## Coverage
- Form 5500 covers ERISA-qualified plans with тЙе100 participants. Smaller plans file Form 5500-SF (separately available).
- "Latest" path is a moving target: DOL revises files as late filings come in. Pull a fresh copy when revisions matter.
- Annual coverage from 1999 onward; full schedule structure is stable from 2009 onward (post-EFAST2 transition).

## How to use

A helper is available at `code/utils/form_5500_utils.py`:
- `download_5500(year)` тАФ main form
- `download_schedule(year, letter)` тАФ letter тИИ {`a`, `c`, `d`, `g`, `h`, `i`, `r`, `mb`, `sb`}
- `link_schedule(main_df, sched_df)` тАФ inner join on `ACK_ID`
- `schedule_h_asset_panel(years)` тАФ stacks Schedule H asset cols across years

```python
from utils.form_5500_utils import download_5500, download_schedule, link_schedule

main22 = download_5500(2022)              # 243K plan filings
sh22   = download_schedule(2022, 'h')     # 100K Schedule H rows (financials)
sr22   = download_schedule(2022, 'r')     # 112K Schedule R rows (retirement)

joined = link_schedule(main22, sh22)      # by ACK_ID (filing-level key)
```

The helper streams downloads with per-chunk timeouts and a wall-clock deadline,
and caches each ZIP + a parquet copy under `data/form_5500/`. Re-runs are
near-instant.

## Key columns

### Identifiers (every file)
| Column | Meaning |
|---|---|
| `ACK_ID` | DOL filing ID тАФ **the right key** when joining schedules to a single filing |
| `EIN` (or `SCH_X_EIN`) | Plan sponsor employer identification number |
| `PN` (or `SCH_X_PN`) | Plan number within a sponsor (a sponsor can run many plans) |
| `(EIN, PN)` | Identifies a *plan* across years; use when building a plan-year panel |

### Schedule H тАФ asset breakdown (158 cols; subset that matters most)
| Column | Meaning |
|---|---|
| `TOT_ASSETS_BOY_AMT` / `EOY_AMT` | Total plan assets (beginning / end of year) |
| `INT_REG_INVST_CO_BOY_AMT` / `EOY_AMT` | **Mutual fund holdings** (registered investment companies) тАФ central for fund-flow / pension-as-shareholder research |
| `INT_COMMON_TR_BOY_AMT` / `EOY_AMT` | Common collective trusts (CCTs) тАФ substitute for mutual funds in DC plans |
| `INT_POOL_SEP_ACCT_*_AMT` | Pooled separate accounts (insurance-wrapped) |
| `INT_MASTER_TR_*_AMT` | Master trust holdings |
| `COMMON_STOCK_*_AMT` | Direct equity holdings |
| `EMPLR_CONTRIB_*_AMT` | Employer contributions |
| `PARTCP_CONTRIB_*_AMT` | Participant contributions |
| `PARTCP_LOANS_*_AMT` | Participant loans outstanding |

Pair `BOY` and `EOY` to construct flows.

### Schedule R тАФ retirement plan info
- Distributions, rollovers, payor identifiers (`PEN_PAYOR_NN_EIN`).
- Use for measuring rollover *outflows* by plan-year.

### Schedule C тАФ service provider compensation
- Per-provider direct + indirect comp; useful for fee-disclosure / record-keeper research.

## Standard operations

- **Plan-year panel.** Loop over years, stack via `schedule_h_asset_panel(range(2010, 2024))`, then keep `(EIN, PN, year)` as the panel key.
- **Mutual-fund exposure share.** `INT_REG_INVST_CO_EOY_AMT / TOT_ASSETS_EOY_AMT` per filing. Aggregate by sponsor, by year, by plan size.
- **Implied flows.** `EOY - BOY * (1 + r_t)` where `r_t` is the plan's realized return on its asset mix (use FF or category benchmarks).
- **DC vs DB.** Filter on plan-type indicators in main 5500 (e.g., `TYPE_PENSION_BNFT_CODE`) тАФ DC plans are where mutual-fund pass-through dominates.
- **Sponsor identification.** Sponsor name + EIN are in main 5500. Match to Compustat by EIN to bring in firm characteristics.

## Performance & rules

- **Cache.** All files cached in `data/form_5500/` automatically. Don't re-download.
- **Big files.** Schedule H is ~17 MB ZIP / ~52 MB CSV per year and ~30 MB parquet. Multi-year panels can reach hundreds of MB in memory тАФ use `usecols` or `SCHEDULE_H_ASSET_COLS` subsets.
- **Slow source.** DOL serves at ~50 KB/s on `urllib`; the helper uses `requests` streaming which negotiates ~2.5 MB/s. Either way, expect a multi-minute first download per (year, schedule).
- **Latest is unstable.** When publishing results, snapshot the cache and report the access date. Re-running months later may yield different row counts as DOL accepts late filings.
- **Always state which schedule, year, and revision.** "Form 5500 Schedule H, plan year 2022, accessed YYYY-MM-DD."
