## New issues

**ISSUE A — Severity: LOW — Placebo battery could be claimed as (c).**
Location: `scorer-core.md` line 28, exception (c).

Exception (c) reads "a formal methodological result with proved properties (e.g., a new estimator or identification theorem)." The trailing "— not an empirical critique of this paper's own design" reads as a parenthetical clarification of the exemption's scope, not as a freestanding second requirement with separate syntax. A savvy agent could argue: "We ran 1,000 placebo iterations and proved a large-sample rejection rate theorem — this is a formal methodological result with proved properties." The examples ("new estimator," "identification theorem") are listed as `e.g.`, not an exhaustive enumeration. The gap: the text does not require that the proved result be *general* (applicable beyond this paper's data) — a proved large-sample rejection rate for this paper's specific design could be dressed as (c). Suggested fix: tighten (c) to "a formal methodological result with proved properties that is general — applicable beyond this paper's specific analysis or dataset (e.g., a new estimator or identification theorem)." This closes the placebo-battery vector without changing (c)'s scope for legitimate methods papers.

**ISSUE B — Severity: MEDIUM — Dataset-release double-listing produces conflicting verdicts under empirical-first for hybrid contributions.**
Location: `scorer-core.md` lines 28 and 33.

"Dataset release" is now listed in both the universal block (trigger: fail unless exception applies) and the empirical-first block (listed in the non-causal drift class, is itself a fail — (a)/(b) do not apply, only (c) does). For a hybrid paper under empirical-first whose contribution is "a new validated dataset PLUS a proved theorem about an estimator built on that dataset": the universal block fires on the dataset-release trigger, then (c) exempts it (estimator theorem has proved properties). The empirical-first block fires on "dataset release" in the drift-class list, then (c) also exempts it (same carve-out). The verdicts are consistent — both blocks end at "exempt via (c)." However, if the same paper's proved estimator theorem is not general (issue A above), neither (c) carve-out saves it, and both blocks condemn it. The double-firing is not a contradiction for (c)-eligible papers, but it is redundant and could confuse a scorer into treating the two blocks as independent filters with an implicit AND. One block failing and the other passing would be treated as FAIL even if (c) is present. The wording "exceptions (a) and (b) above do not apply here" in the empirical-first block uses "above" to refer to the universal block's (a)/(b), implying shared exception vocabulary — a scorer reading carefully will see they share (c) too. The risk is low but non-zero. Suggested direction: in the empirical-first block, replace the parenthetical list of drift classes with "drift to a non-causal contribution class ... is itself an H1/H5 FAIL" and note "(c)'s exemption applies to both blocks identically — if (c) saves the paper in the universal guard, it saves it here too." This makes the shared exception semantics explicit and prevents double-jeopardy readings.

## Walkthroughs

**Placebo battery paper (1,000 iterations, 94.9% rejection rate).** The scorer reads the own-design-critique guard: the contribution is a "methodological warning" that this paper's own placebos fail — trigger fires. The agent asks: does (c) apply? A rejection-rate measurement is not a proved theorem about an estimator's properties. Goodman-Bacon's decomposition is proved algebra; a simulation rejection rate is a statistic. Exception (c) does not apply. Exceptions (a) and (b) are absent. H1/H5 FAIL — correct outcome. No slip-through under current text, but the gap opens if an agent calls the rejection rate a "proved large-sample result."

**"New estimator with proved consistency" under empirical-first.** Universal block fires on "methods note / formal estimator" but exempts via (c). Empirical-first block fires on causal-estimand drift if the estimator contribution displaces a causal identification claim. Exception (c) is explicitly reinstated in the empirical-first block. Net verdict: both blocks exempt the paper. H1/H5 PASS on this vector — the paper still must pass H2-H4 and get scored. Correct outcome; fix 1 holds under this walkthrough.

**Hybrid dataset-release + estimator theorem under empirical-first.** Universal block trigger fires (dataset release is claimed contribution). (c) exempts if the estimator theorem has proved properties. Empirical-first block fires on "dataset release" in the drift-class list. (c) still applies per the carve-out. Consistent exemption — the paper passes the guard. But: if the proved theorem is not general (issue A), (c) fails in both blocks simultaneously, and the paper gets H1/H5 FAIL from both guards. This is the correct outcome. The double-listing is redundant but not contradictory for the well-formed cases. Redundancy becomes a problem only if a scorer misreads the blocks as independent ANDs with different (c) scopes — which the current wording does not clearly prevent.
